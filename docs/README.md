## Local Development

### Prerequisites

- Node.js 18 or later
- npm (comes with Node.js)

### Installation

```bash
cd docs/website
npm install
```

### Start Development Server

```bash
npm start
```

This will open http://localhost:3333 with live reload (port 3333 avoids conflict with A2A agent server on port 3000).

### Build for Production

```bash
npm run build
```

The static files will be generated in `docs/website/build/`.

## Deployment

Documentation is automatically deployed to GitHub Pages when changes are pushed to the `main` branch.

The deployment is handled by `.github/workflows/deploy-docs.yml`.

### Manual Deployment

If needed, you can deploy manually:

```bash
cd docs/website
npm run build
npm run serve  # Test the build locally first
# Then deploy via GitHub Actions or manually upload build/ directory
```

## Configuration

### docusaurus.config.ts

Main configuration file controlling:
- Site metadata (title, tagline, URL)
- Theme configuration
- Plugins (search, sitemap, etc.)
- Navbar and footer

### sidebars.ts

Controls the documentation sidebar structure. Uses autogenerated sidebars based on file hierarchy and frontmatter `sidebar_position`.

## Adding Documentation

### Creating New Pages

1. Create a Markdown file in `content/` in the appropriate directory
2. Add frontmatter:
   ```markdown
   ---
   sidebar_position: 1
   ---
   
   # Page Title
   
   Content here...
   ```

3. The page will automatically appear in the sidebar based on its location and `sidebar_position`

### Updating Homepage

Edit `website/src/pages/index.tsx` to customize the landing page.

### Custom Styling

Edit `website/src/css/custom.css` to customize colors, fonts, and other styles.

## Assets

### Logo and Favicon

Currently using placeholder assets:
- `website/static/img/logo.svg` - Placeholder logo
- `website/static/img/favicon.ico` - Placeholder favicon

Replace these with actual branded assets when available.

### Images

Place images in:
- `website/static/img/` for general images
- `content/<section>/images/` for section-specific images

Reference in Markdown:
```markdown
![Alt text](/img/my-image.png)
```

## Troubleshooting

### Port Already in Use

The dev server uses port 3333 by default (port 3000 is reserved for the A2A agent server). If port 3333 is also in use:

```bash
npm start -- --port 3334
```

### Build Errors

```bash
npm run clear  # Clear cache
npm install    # Reinstall dependencies
npm run build  # Try build again
```

### Broken Links

Docusaurus will warn about broken internal links during build. Fix them before deploying.
